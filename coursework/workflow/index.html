Разработка серверной части:

Использованные технологии: Django, Django REST framework.

В ходе разработки были реализованы: * Модель базы данных средствами DjangoORM * Логика работы API средствами Django REST Framework * Регистрация / авторизация по токенам / вывод информации о текущем пользователе средствами Djoser

Разработка клиентской части:

Использованные технологии: Vue JS, Vuetify.

В ходе разработки было настроено взаимодействие с сервером через CORS (Cross-origin resource sharing), а также реализованы следующие интерфейсы: * Авторизация, регистрация и изменение учётных данных * Создание, вывод, изменение и удаление данных в БД

Лабораторная работа 1. Контейнеризация и оркестрация.

В данной работе с помощью ПО Docker весь код сайта (бэкенд, фронтенд), а также его БД были перенесены в контейнеры. Каждый из контейнеров отвечает за свою часть приложения, доступ содержимого одного контейнера к содержимому другого возможен благодаря сети, соединяющей контейнеры.

Лабораторная работа 2. Возможности Django REST Framework.

В данной работе была реализована фильтрация объектов (ручная и с помощью django-filter), настроена стандартная пагинация на весь проект и кастомная для определенного представления, реализованы эндпойнты для загрузки и валидации файлов, и написаны несколько сигналов.

Лабораторная работа 3. Тесты Django REST Framework.

В данной работе были реализованы модульные тесты (на модель, GET, POST и PATCH запросы). В разработке использовался нативный фреймворк Django для создания тестов, основанный на стандартной библиотеке python unittest.

Лабораторная работа 4. Адаптация интерфейсов vue.js.

В данной работе было реализовано взаимодействие с созданными в ЛР2 эндпойнтами для сортировки, фильтрации и поиска объектов, а также настроен приём пагинированных ответов с сервера.
